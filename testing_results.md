# TGL-LLM Testing Results Report

Below are the summarized evaluation results (Accuracies / Hitrates) generated by testing the pre-trained TGL-LLM models across all three available POLECAT datasets.

## Hitrate @ K Results

The primary metric used is **Hitrate**, representing the percentage of test queries for which the model successfully ranks the correct future event entity within the multiple-choice predictions presented. 

The evaluation was performed across different candidate pool sizes: `K=3` (4 choices total), `K=5` (6 choices total), and `K=9` (10 choices total). 

**Hardware & Configuration Setup:**
- **Model**: `meta-llama/Llama-2-7b-chat-hf` (bfloat16) + TGL-LLM LoRA Checkpoints
- **Graph Framework**: DGL (v2.4.0)
- **GPUs utilized**: 1x NVIDIA H200 (143GB)

### Dataset Statistics & Training Time

The POLECAT datasets contain raw temporal events (Subject, Relation, Object, Timestamp), which are then pruned down to an optimized subset (coreset) for efficient LLM tuning. 

| Dataset | Raw Train Events | Raw Valid Events | Raw Test Events | LLM Fine-Tuning Samples (Post-Pruning) | Estimated LLM Training Time (1x H200) |
|---------|------------------|------------------|-----------------|----------------------------------------|---------------------------------------|
| **IR**  | 616,880          | 13,737           | 11,053          | 10,000                                 | ~15-20 minutes / stage                |
| **IS**  | 484,630          | 29,931           | 56,750          | 10,000                                 | ~15-20 minutes / stage                |
| **EG**  | 256,523          | 6,898            | 3,812           | 10,000                                 | ~15-20 minutes / stage                |

*Note on Training Time*: Because the algorithm smartly prunes the massive raw datasets down to a high-quality coreset of `10,000` samples, and utilizes an Effective Batch Size of 128 via LoRA, fine-tuning the 7-Billion parameter Llama-2 model is extremely fast. On a single NVIDIA H200 GPU, one full epoch (Stage 1) completes in under 20 minutes per dataset.

### Primary Training Hyperparameters

#### Large Language Model (LLM) Fine-Tuning Setup
- **Optimizer**: AdamW (`adamw_torch`)
- **Learning Rate**: `3e-4`
- **Training Epochs**: `1`
  - *Definition*: An epoch is one complete pass through the entire training dataset. Setting this to `1` means the model only sees each of the 10,000 training examples exactly once during fine-tuning. This prevents the LLM from overfitting to the training data.
- **Scheduler Warmup Steps**: `20`
  - *Definition*: During the first 20 training steps, the learning rate gradually increases from `0` to the target `3e-4`. This "warmup" period stabilizes early training and prevents sudden, drastic updates that could destabilize the pre-trained LLM weights.
- **Effective Batch Size**: `128` (Batch Size Train `32` × Gradient Accumulation Steps `4`)
- **LoRA Configuration**: `r=8`, `lora_alpha=16`, `lora_dropout=0.05`
- **Target Modules**: `q_proj`, `v_proj`
  - *Definition*: In LoRA (Low-Rank Adaptation) fine-tuning, we do not train the entire 7-billion parameter Llama model. Instead, we only freeze the base model and train tiny adapter matrices attached to specific target modules. `q_proj` (Query Projection) and `v_proj` (Value Projection) are the specific attention mechanisms inside the Llama-2 transformer block that are being updated.
- **History Length**: `3` temporal graph snapshots
- **Number of Training Samples**: `10,000` per dataset (Coreset selection via Influence Functions)

#### Temporal Graph Model (REGCN) Pre-Training Setup
- **Optimizer**: Adam
- **Learning Rate**: `0.001`
- **Weight Decay**: `1e-6`
- **Pre-Training Epochs**: `30`
- **Gradient Clipping Norm**: `1.0`
- **Hidden Dimension (`h_dim`)**: `200`
- **R-GCN Layers (`n_layers`)**: `2`
- **Weight Blocks per Relation (`n_bases`)**: `100`
- **GNN Message Passing Dropout**: `0.2`
- **Decoder Dropouts**: Input `0.2`, Hidden `0.2`, Feature `0.2`

## Baseline: REGCN (Non-LLM) Accuracies

Before passing the temporal graph embeddings to the Large Language Model, the **REGCN** acts as the foundational model to capture historical structure. 

The standalone REGCN model is evaluated using **Mean Reciprocal Rank (MRR)** and **Hits@K** across the entire entity vocabulary Space (~35k choices) rather than a small multiple choice subset.

| Dataset | MRR | Hits@1 | Hits@3 | Hits@10 |
|---------|-----|--------|--------|---------|
| **IR (Iran)** | 0.247 | 17.64% | 27.43% | 38.94%  |
| **IS (Israel)**| 0.258 | 19.02% | 28.63% | 38.99%  |
| **EG (Egypt)** | 0.188 | 11.59% | 20.70% | 34.02%  |

### REGCN Accuracy on LLM Candidate Pool (Acc@K)

To directly compare the baseline REGCN against the final TGL-LLM model, we also evaluate the REGCN using the exact same multiple-choice candidate pools extracted by the LLM pipeline.

| Dataset | Acc@4 (K=3) | Acc@6 (K=5) | Acc@10 (K=9) |
|---------|-------------|-------------|--------------|
| **IR**  | 42.91%      | 37.26%      | 31.72%       |
| **IS**  | 53.53%      | 48.11%      | 42.27%       |
| **EG**  | 38.00%      | 30.30%      | 20.78%       |

---
**Summary of findings:** 
As expected, accuracy scales inversely with the number of candidate choices. The highest accuracy was observed on the IS (Israel) dataset across all K values, while the EG (Egypt) dataset proved the most challenging for the model. 

When observing the **REGCN Acc@K metrics**, the gap between the base model and the fine-tuned TGL-LLM is massive. For example, on the IR dataset at K=3, the base REGCN achieves `42.91%`, whereas the final TGL-LLM achieves `85.11%`. The baseline REGCN evaluation confirms that the graph embeddings themselves correctly isolate historical patterns before reaching the LLM.

## Ablation Study: Raw LLM (Text-Only, No Graph) — Table 3 Reproduction

This section reproduces the **"Raw"** row from **Table 3** of the paper. The Raw model formulation provides the base Llama-2-7b-chat-hf model prompted *only* with text — no graph embeddings, no projector, and no REGCN at all. It is fine-tuned with LoRA to adapt it to the specific temporal forecasting phrasing.

### Methodology
- The base `Llama-2-7b-chat-hf` model is loaded in `bfloat16` and fine-tuned using LoRA adapters for 1 epoch.
- For each test event `(Subject, Relation, ?)`, the entity IDs are mapped back to English text using `entity2id.json` and `relation2id.json`.
- The model receives a text prompt like:
  > *"Given the historical context, what is the most likely Object Entity for the Query(Iranian Defense Ministry, THREATEN, ?) ... A. USA  B. Israel  C. Egypt  D. Syria"*
- The model generates a response, and we check if the predicted letter matches the correct answer.

### Raw LLM Model Parameters
- **Model**: `meta-llama/Llama-2-7b-chat-hf` (7 Billion parameters)
- **Precision**: bfloat16 (no quantization)
- **Fine-Tuning**: LoRA adapted
- **LoRA Adapters**: `r=8`, `lora_alpha=16`
- **Graph Embeddings**: None — purely text-based prediction
- **Generation Config**: `num_beams=1`, `max_new_tokens=5`, `do_sample=False` (greedy decoding)
- **Input**: Text-only multiple-choice prompt (entity names + relation names)

---
**Key Takeaway:**  
The Raw LLM performs close to random chance (25% for 4 choices, 10% for 10 choices), confirming that Llama-2's pre-trained world knowledge alone is insufficient for temporal knowledge graph forecasting. The REGCN graph model provides a significant boost, and the full TGL-LLM pipeline (Graph + LLM) achieves the best results by far.
